# user-crud-api

Short description
A NestJS REST API for user CRUD + authentication (TypeORM + MySQL, JWT auth). Provides endpoints for user registration, login/refresh/logout and basic user CRUD.

Features
- NestJS + TypeORM
- MySQL integration
- JWT authentication (Passport)
- Refresh token support
- Docker optional (if present)

Prerequisites
- Node.js >= 18
- npm or pnpm
- MySQL database
- (optional) GitHub CLI `gh` for repo creation
- (optional) ngrok or VS Code Dev Tunnels for public exposure

Environment variables
Create a `.env` file in project root with (example):
```
DB_HOST=localhost
DB_PORT=3306
DB_USER=root
DB_PASS=your_db_password
DB_NAME=userdb
JWT_SECRET=your_jwt_secret
JWT_EXPIRES_IN=3600s
REFRESH_TOKEN_EXPIRES_IN=7d
PORT=3000
```

Install & run (local)
```powershell
npm install
# development (watch)
npm run start:dev
# build
npm run build
# production
npm run start:prod
```

Database
- Ensure MySQL is running and DB (DB_NAME) exists or let TypeORM create/update schema depending on config.
- If migrations are used, run the migration commands in package.json (e.g. `npm run typeorm:migrate`).

Expose via Dev Tunnel (VS Code Dev Tunnels)
1. Ensure app listens on 0.0.0.0 (see src/main.ts).
2. Start app: `npm run start:dev`
3. In VS Code: Command Palette → "Dev Tunnels: Start Public Tunnel" → select port 3000.
4. Copy public URL and test.

Create GitHub repo and push (example)
Replace placeholders before running:
```powershell
# initialize
git init
git add .
git commit -m "Initial commit"

# create remote & push using GitHub CLI (recommended)
gh auth login
gh repo create <your-username>/<repo-name> --public --source=. --remote=origin --push

# or manual (create repo on GitHub, then:)
git branch -M main
git remote add origin https://github.com/<your-username>/<repo-name>.git
git push -u origin main
```
